# Documentation guidelines

- **Match the project's docs.** Follow the existing format, tone, and location
  (README, docs/, doc comments); don't introduce a parallel system.
- **Explain the why.** Document intent, rationale, and non-obvious constraints —
  the code already shows the what.
- **Keep docs in sync.** Update the relevant docs/README/comments in the same
  change that alters the behavior or interface they describe.
- **Don't document unprompted.** Don't create new standalone docs (READMEs,
  summaries) unless asked or the project clearly expects them.
